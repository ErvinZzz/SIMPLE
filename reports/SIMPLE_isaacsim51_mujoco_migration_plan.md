## Target versions

## Summary
Migration plan (PLAN ONLY — no repo edits) to move SIMPLE @ d010f0f from IsaacSim 4.5.0 + MuJoCo 3.3.6 + Python 3.10 to IsaacSim 5.1.0 + MuJoCo 3.9.0 + Python 3.11, required by the RTX 5090 (Blackwell sm_120), which has no support before IsaacSim 5.0. Two coupled workstreams. (A) IsaacSim: a version+Python bump that MUST land together with a code-side import migration from the deprecated omni.isaac.* namespace to isaacsim.* — a pure version bump without the import migration will ImportError because in 5.x the omni.isaac.* extensions exist only as deprecation shims under isaacsim/extsDeprecated/ and are slated for full removal in 6.0. The renaming itself happened in IsaacSim 4.5 (extensions_renaming.html); 5.0/5.1 keep only the shims. (B) MuJoCo: a pure version bump (3.3.6 -> 3.9.0); SIMPLE's entire MuJoCo surface (MjSpec/MjModel/MjData/mj_step/mj_forward/Renderer/viewer/mjt* enums) is unchanged across 3.3.6->3.9.0 — all 3.4-3.9 breaking changes are at the C/mjs-builder/MJX layer SIMPLE does not touch. The only MuJoCo work is BEHAVIORAL smoke-tests (multiccd default-on at 3.8.0, margin/gap redesign at 3.9.0, texture colorspace). The 5.0-era "runtime-added prims do not render under Fabric" symptom is addressed in 5.1 by (a) the new isaacsim.core.api boolean flag to force fabric update on a physics-only step, (b) the XFormPrim get/set_world_poses Fabric Scene Delegate rewrite, and (c) the /Render prim visibility workaround removal — but NONE of these is a documented fix for that exact symptom, so it MUST be verified empirically after migration. SIMPLE already uses the non-tiled Camera class, putting it on the safe side of the open RTX 5090 TiledCamera hang (IsaacLab #4951). The omni.replicator.core import path and the already-migrated isaacsim.util.debug_draw / isaacsim.core.prims.SingleArticulation imports need no change. Per-class isaacsim.* sub-paths (World/Robot/objects under isaacsim.core.api) are not in the renaming table at class granularity; they were verified on-disk against the installed isaacsim 5.0.0.0 module tree (same layout as 5.1) and should be re-pinned against the installed 5.1 tree before editing. NEVER pip-install into a conda env (a prior agent broke env_isaaclab by installing usd-core — pxr is bundled inside the Kit wheels, not a separate pip dist).
